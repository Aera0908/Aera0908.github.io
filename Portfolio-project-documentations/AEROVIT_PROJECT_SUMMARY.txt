(Compiled from web-showcase and web-doc)

============================================================
1) SYSTEM SUMMARY
============================================================
Aerovit is a hybrid fitness ecosystem that combines:
- A mobile fitness app (Flutter/Dart)
- Computer vision pose tracking (MediaPipe BlazePose)
- A custom ESP32-S3 smartwatch for biometric sensing and BLE streaming
- Gamified progression (ranking, quests, dungeon RPG)
- Web3 tokenization (AERO token on Ethereum Sepolia)
- AI coaching systems (RAG + pre-recorded audio + RL adaptation)

Core value proposition:
Users complete workouts and quests, receive biomechanic and biometric feedback,
progress through hunter ranks, and earn in-game/off-chain rewards that can be
withdrawn on-chain as ERC-20 AERO tokens.

4) CORE FEATURE DOMAINS
============================================================
4.1 Gamification & Progression
- Hunter rank progression: E, D, C, B, A, S, National
- XP from workouts, reps, form quality, quests, streaks, achievements
- Daily quests with reset cycle and streak multipliers
- Social leaderboards and friend comparison
- Trophy case and unlockable profile titles

4.2 Pose Detection & Exercise Intelligence
- MediaPipe BlazePose (33 landmarks, 3D coordinates + visibility)
- Real-time form validation and color-coded angle feedback
- Rep counting using phase/state logic and angle thresholds
- EMA-based smoothing and low-latency processing pipeline
- Multi-exercise support (22+ to 24 exercise coverage referenced)

4.3 AI Coaching
- RAG workout guidance using curated fitness corpus (ChromaDB)
- Voice/text interactions with STT/TTS integration
- Pre-recorded low-latency audio coaching with modular phrase chaining
- Multiple coach personalities (serious/friendly/fierce)
- RL adaptation for workout intensity and intervention strategy

4.4 Wearable + Biometric Intelligence
- Real-time HR/SpO2 capture and watch-to-app BLE streaming
- Step counting via high-pass filter + zero-crossing detection
- Cadence-based activity classification (6 tiers)
- Danger detection for abnormal HR/SpO2 and overexertion conditions
- Max-HR session tracking and calorie estimation (MET-based)
- Smart wake/sleep behavior and battery-aware UI flow

4.5 Dungeon Raid RPG
- 20-floor turn-based dungeon system
- 10 enemy species + 2 major boss milestones
- 23 skills across physical/magic/heal/buff/status groups
- 10 status effects for strategic combat
- Equipment system: 11 weapons + 11 armors + rarity tiers
- Consumable inventory and shop economy
- Save/resume and reward synchronization with user progression

4.6 Web3 + Token Economy
- AERO token (ERC-20, Sepolia testnet)
- Move-to-earn reward conversion from fitness activity
- Off-chain accumulation with on-chain withdrawal
- Deflationary burn mechanism (5% withdrawal burn)
- Hardware wallet-assisted signing flow via watch

6) BIOMETRIC PROCESSING DETAILS
============================================================
6.1 Step Detection Parameters
- STEP_THRESHOLD: 0.15g
- MIN_INTERVAL: 250ms
- MAX_INTERVAL: 2000ms
- HP_ALPHA: 0.96

High-pass filter model:
hp_output = alpha * (hp_output_prev + raw - raw_prev)

6.2 Activity Classification Bands (cadence SPM)
- Idle: 0-14
- Slow Walk: 15-79
- Normal Walk: 80-99
- Brisk Walk: 100-119
- Jogging: 120-149
- Running: 150+

6.3 Danger Detection Thresholds
- High HR: > 180 BPM
- Low SpO2: < 90%
- Overexertion condition: HR > 170 and SpO2 < 93%

6.4 MAX30102 Notes
- HR valid range filtering (40-180 BPM noted)
- SpO2 ratio-based estimation and calibrated lookup methods
- Finger detection hysteresis to reduce reading flicker

11) HARDWARE WALLET SECURITY MODEL
============================================================
Documented security elements include:
- On-device key generation and storage
- BIP-39/BIP-32 style derivation references
- secp256k1 ECDSA transaction signing
- AES-256 encryption at rest (NVS storage)
- PIN protection with PBKDF2 and lockout/cooldown
- BLE operation commands for wallet lifecycle actions
- Private key usage constrained to signing windows and memory wipe behavior

12) USER FLOW (END-TO-END)
============================================================
1. User installs app and signs in
2. User configures profile/fitness settings
3. User pairs watch via QR + BLE
4. User starts workout
5. Pose pipeline tracks reps/form in real time
6. Watch streams HR/SpO2/activity/steps
7. AI coach + RL adaptation provide feedback/interventions
8. Quest/XP/rank progress updates and persists
9. Optional dungeon gameplay uses fitness-linked progression
10. User can withdraw earned rewards through wallet flow
