1) PROJECT OVERVIEW
-------------------
Manhwa Reader is a modern web application built to browse and read manhwa chapters using the MangaDex API.
The app focuses on a clean reading experience, responsive layout, dark/light theme support, and simple navigation.

Core user flow:
Home -> Manhwa Detail -> Chapter Reader


2) PROBLEM IT SOLVES
--------------------
- Helps readers discover popular and recently updated manhwa.
- Provides fast title-based search.
- Allows users to open chapter lists and read pages in a dedicated reader view.
- Offers accessible UI options with dark/light mode.


3) MAIN FEATURES
----------------
- Home page with:
  - Hero search input
  - Popular manhwa section
  - Recently updated section
  - Category chips (UI)
- Search function:
  - Query-based search against MangaDex
  - Result grid with quick access to details
- Detail page:
  - Cover, title, latest chapter, synopsis
  - English chapter list
  - Start reading button
- Reader page:
  - Full chapter image loading
  - Chapter selector dropdown
  - Previous/Next chapter navigation
  - Optional control panel toggle
- Settings page:
  - Light/Dark theme switch
  - UI placeholders for reading preferences and notifications
- Error and empty states:
  - Loading spinners
  - "Not found" and "No pages/chapters" fallbacks

6) ROUTING MAP
--------------
- /                      -> Home
- /settings              -> Settings
- /manhwa/:id            -> Manhwa detail page
- /read/:mangaId/:chapterId -> Reader page


7) API INTEGRATION NOTES
------------------------
The app integrates with MangaDex endpoints to:
- Fetch popular titles
- Fetch recently updated titles
- Search manga by title
- Fetch manga metadata by ID
- Fetch chapter feed (English)
- Resolve chapter image server + page URLs

Important behavior:
- Chapters are paginated in batches and merged.
- Duplicate chapter numbers are filtered; better page-count entries are preferred.
- Chapter pages are built from baseUrl + chapter hash + filename.

10) CHALLENGES & SOLUTIONS (PORTFOLIO TALKING POINTS)
-----------------------------------------------------
Challenge:
- MangaDex data can have missing fields and inconsistent chapter uploads.

Solution:
- Added robust fallbacks for title, description, and chapter labels.
- Implemented duplicate chapter filtering logic.
- Added loading and empty/error UI states to keep user experience stable.

Challenge:
- Reading view needs smooth chapter switching and quick controls.

Solution:
- Built in-reader chapter selector plus prev/next actions.
- Added control visibility toggle for cleaner page viewing.
